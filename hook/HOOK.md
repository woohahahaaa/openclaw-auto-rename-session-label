# Auto Rename Session Label

On `message:received`, gives a not-yet-labeled session a short title generated by the **SAME model** the session is currently using. Falls back to truncating the first user message on any failure. Portable — resolves all paths at runtime, contains no hardcoded machine paths or personal data.

## Rules

1. Filter `message:received`; read the corresponding `sessionKey` entry from `sessions.json`.
2. If `label` already exists and is non-empty → **skip**.
3. Get the first user message: prefer `event.context.content` (instant), else read `.jsonl` transcript.
4. **Call LLM for title**: Use the session's current model (read `modelOverride`/`model` + `providerOverride`/`modelProvider`, map to real provider id via `models.providers` table). Reuse OpenClaw's internal `completeSimple` chain for a one-shot completion.
   - Instruction in English; title language follows user's original message.
   - Instruction embedded in user message (some upstream proxies reject system messages).
   - Timeout 15s. Clean quotes/newlines, hard‑cap 30 chars.
5. **Fallback**: If model unavailable, resolution fails, timeout, or upstream error → revert to "first 50 chars of first message + …". Guarantees a label is always written.
6. Re‑read `sessions.json` just before write to avoid clobbering concurrent updates. Silent execution, no replies.

## Implementation notes

- Uses `handler.js` (plain JavaScript, no TypeScript compilation needed).
- Dynamically discovers OpenClaw home, agent id, and dist directory.
- Locates hashed dist modules (`model-*.js`, `stream-*.js`, etc.) by exported‑symbol name, resilient to version bumps.
- Concurrent‑safe: reads fresh copy of sessions.json before writing.
